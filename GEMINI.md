# Обзор проекта "Tasker / TaskFlow"

## Обзор

Это платформа для AI-агента с веб-интерфейсом. Выглядит как продвинутый инструмент для управления проектами.

*   **Фронтенд:** Создан с использованием Next.js и Prisma. Включает в себя Kanban-доску, календарь и представление в виде временной шкалы.
*   **Бэкенд:** Состоит из модульных "навыков" (skills) искусственного интеллекта, таких как обработка документов, генерация изображений и веб-поиск. Эти навыки, похоже, спроектированы как независимые микросервисы.
*   **Архитектура:** Файл `Caddyfile` указывает на использование обратного прокси-сервера для управления маршрутизацией между веб-приложением и различными AI-микросервисами.

## Ключевая цель

Предоставить удобный интерфейс для управления мощным AI-агентом, способным выполнять и координировать широкий спектр задач.

## Ключевые файлы

*   `prisma/schema.prisma`: Определяет структуру данных (модели проектов, задач, пользователей).
*   `skills/`: Содержит код для различных AI-возможностей.
*   `src/app/(dashboard)/projects/[projectId]/page.tsx`: Основная страница для просмотра и управления конкретным проектом.
*   `src/components/ai/ai-assistant.tsx`: Пользовательский интерфейс для взаимодействия с AI-ассистентом.
*   `Caddyfile`: Конфигурация для развертывания и маршрутизации.

## Анализ проекта (результаты глубокого изучения)

### Технологический стек

*   **Фреймворк:** Next.js (с React)
*   **UI:** shadcn/ui, Tailwind CSS, Recharts (для графиков), dnd-kit (для drag-and-drop).
*   **Управление состоянием:** Zustand и TanStack Query.
*   **База данных:** Prisma ORM (с SQLite).
*   **Аутентификация:** NextAuth.js.
*   **Инструменты:** TypeScript, ESLint, Bun.

### Архитектура данных

Схема `prisma/schema.prisma` описывает комплексную модель для управления проектами, включающую:
*   **Пользователи (`User`)**: Управление аккаунтами и аутентификацией.
*   **Проекты (`Project`)**: Основная сущность с владельцами, участниками и иерархией.
*   **Задачи (`Task`)**: Единицы работы с подзадачами, зависимостями и статусами.
*   **Шаблоны (`Template`)**: Для создания переиспользуемых структур проектов.
*   **Вспомогательные модели**: `Comment`, `Tag`, `Attachment` и другие для поддержки полной функциональности.

### Ключевые компоненты

*   `src/app/(dashboard)/projects/[projectId]/page.tsx`: Центральная страница для просмотра проекта. Она динамически загружает данные и позволяет пользователю переключаться между разными видами (доска, таймлайн, список, календарь).
*   `src/components/ai/ai-assistant.tsx`: Компонент AI-ассистента. **На данный момент это UI-макет.** Вся логика "ИИ" симулируется на стороне клиента с помощью `setTimeout` и заготовленных ответов. **Интеграция с реальными AI-моделями или микросервисами из папки `/skills` отсутствует.**

### Общий вывод

Проект "Tasker / TaskFlow" представляет собой хорошо спроектированную основу для системы управления проектами с современным технологическим стеком. Основная функциональность по управлению задачами реализована и работает.

Ключевая особенность — **AI-ассистент** — в настоящее время является **прототипом (mockup)**. Следующим логическим шагом в развитии проекта должна стать интеграция этого ассистента с реальным бэкендом (AI "навыками"), чтобы реализовать заявленные интеллектуальные возможности.
